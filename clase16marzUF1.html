1. ¿Qué es el despliegue?

El despliegue de aplicaciones web o deployment es el conjunto de procesos técnicos necesarios para poner una aplicación web en funcionamiento dentro de una infraestructura accesible para los usuarios.

Incluye

  • Instalar la aplicación.
  • Configurar servidores.
  • Preparar bases de datos.
  • Configurar dominios.
  • Garantizar seguridad y rendimiento.
Idea clave: desplegar no es programar, sino dejar la aplicación funcionando para que los usuarios puedan usarla.

2. El despliegue dentro del ciclo de vida del software

Fase Descripción
Análisis Se identifican necesidades y requisitos.
Diseño Se planifica la arquitectura del sistema.
Implementación Se desarrolla el código.
Implantación / Despliegue Se instala el software en servidores.
Mantenimiento Se corrigen errores y se actualiza.
Muy importante: el despliegue se realiza después de la implementación, cuando la aplicación ya está desarrollada y probada.

3. Características de las aplicaciones web

  • Se ejecutan en servidores web.
  • Atienden múltiples usuarios simultáneamente.
  • Funcionan a través de Internet.
  • Requieren disponibilidad continua.

En sistemas grandes pueden atender miles o millones de usuarios, por eso se usan sistemas distribuidos y clústeres de servidores.

4. Escalabilidad

La escalabilidad es la capacidad de un sistema para aumentar su capacidad de servicio cuando crece la demanda.

Escalado vertical

Consiste en aumentar los recursos del servidor existente.

  • Más RAM
  • Más CPU
  • Más almacenamiento
Ventajas: fácil de implementar y no cambia la arquitectura.
Desventajas: tiene límite físico y suele ser más caro.

Escalado horizontal

Consiste en añadir más servidores al sistema.

Configuración Descripción
1 servidor Sistema simple
3 servidores Mayor capacidad
10 servidores Alta disponibilidad

Se apoya en balanceadores de carga, replicación de bases de datos y microservicios.

Ejemplo real: Netflix utiliza miles de servidores distribuidos en diferentes centros de datos para soportar millones de usuarios, evitar caídas y repartir la carga.

5. Infraestructura para el despliegue

Existen dos formas principales de alojar aplicaciones web:

  • Infraestructura propia
  • Computación en la nube

Infraestructura propia (in-house)

La organización gestiona y mantiene su propia infraestructura informática dentro de sus instalaciones.

Esto implica disponer de:

  • Servidores físicos
  • Almacenamiento
  • Red interna
  • Sistemas de seguridad
  • Sistemas de respaldo
Servicio Función
DNS Resolución de nombres de dominio
LDAP / Active Directory Autenticación de usuarios
Servidores web Ejecución de aplicaciones
FTP Transferencia de archivos
Correo electrónico Comunicación interna
SSH / RDP Administración remota
Ventajas: control total, personalización y seguridad interna.
Desventajas: alto coste, mantenimiento complejo y necesidad de personal técnico.

Caso real: centros de datos

Muchas universidades y administraciones públicas utilizan centros de datos propios con racks de servidores, climatización, sistemas eléctricos redundantes y redes internas de alta velocidad.

6. Computación en la nube

La computación en la nube o cloud computing es un modelo de prestación de servicios informáticos donde los recursos se proporcionan a través de Internet bajo demanda.

  • Pagar solo por lo que se usa.
  • Escalar recursos fácilmente.
  • Evitar inversiones iniciales en hardware.
Origen del cloud moderno: en 2006 Amazon lanzó EC2 (servidores virtuales) y S3 (almacenamiento).

7. Modelos de servicios cloud

IaaS (Infrastructure as a Service)

El proveedor ofrece infraestructura virtual y el usuario gestiona:

  • Sistema operativo
  • Aplicaciones

Ejemplos: Amazon EC2, Azure Virtual Machines, Google Compute Engine.

PaaS (Platform as a Service)

El proveedor gestiona hardware, sistema operativo y middleware. El desarrollador solo sube su aplicación.

Ejemplos: Heroku, Google App Engine, Firebase.

SaaS (Software as a Service)

El proveedor ofrece aplicaciones completas accesibles por Internet. El usuario solo utiliza el software.

Aplicación Uso
Gmail Correo

Modelo Qué gestionas tú
IaaS Sistema operativo y aplicaciones

9. DNS (Domain Name System)

El DNS es un sistema distribuido que permite traducir nombres de dominio en direcciones IP.

Estructura jerárquica

El DNS se organiza como un árbol invertido.

.
|
com
|
google.com
|
www.google.com

Dominios y subdominios

  • Dominios de segundo nivel: google.com, amazon.com, mheducation.es
  • Subdominios: www.google.com, mail.google.com, api.google.com

Resolución DNS

  • Resolución directa: nombre → IP
  • Resolución inversa: IP → nombre
Ejemplo: google.com → 142.250.184.68

Ejercicio DNS: NovaEmpresa

Dominio principal: novaempresa.com

Subdominios de servicios:

  • www.novaempresa.com
  • mail.novaempresa.com
  • ftp.novaempresa.com
  • cloud.novaempresa.com
  • api.novaempresa.com

Subdominios de departamentos:

  • dev.novaempresa.com
  • sys.novaempresa.com
  • marketing.novaempresa.com
.
└── com
    └── novaempresa.com
        ├── www
        ├── mail
        ├── ftp
        ├── cloud
        ├── api
        ├── dev
        ├── sys
        └── marketing
Clave: todos los subdominios cuelgan directamente del dominio principal; no se separan en ramas de “servicios” y “departamentos” dentro del árbol DNS salvo que el enunciado lo pida.

10. LDAP

LDAP (Lightweight Directory Access Protocol) es un protocolo que permite gestionar directorios de información de usuarios y recursos dentro de una red.

Funciones principales

  • Autenticación de usuarios
  • Gestión de permisos
  • Directorios corporativos
  • Correo electrónico
  • Acceso a aplicaciones

Funcionamiento

LDAP sigue un modelo cliente-servidor:

Usuario
↓
Aplicación
↓
Servidor LDAP
↓
Autenticación

Estructura jerárquica: DIT

La información se organiza en un árbol llamado DIT (Directory Information Tree).

dc=empresa
 ├── ou=usuarios
 │     ├── uid=juan
 │     └── uid=ana
 └── ou=grupos
       ├── admins
       └── users
Idea clave: DNS organiza nombres de dominio; LDAP organiza usuarios y permisos.

11. Ejercicios de dominio

Ejercicio 1: Tienda online

Entidades principales: Usuario, Perfil, Pedido, Producto, Categoría y Pago.

TiendaOnline
 ├── Usuario
 │     └── Perfil
 └── Pedido
       ├── Producto
       │     └── Categoría
       └── Pago

Relaciones importantes:

  • Usuario → Pedido
  • Pedido → Producto
  • Producto → Categoría
  • Pedido → Pago

Ejercicio 2: Plataforma de cursos

Entidades principales: Usuario, Profesor, Alumno, Curso, Lecciones, Evaluaciones y Matrícula.

PlataformaEducativa
 └── Usuario
       ├── Profesor
       │     └── Curso
       │           ├── Lecciones
       │           └── Evaluaciones
       └── Alumno
             └── Matrícula
                   └── Curso
Método para estos ejercicios: primero identifica sustantivos del enunciado como posibles entidades, luego detecta relaciones y por último construye el árbol.

12. Claves de examen

  • Despliegue = poner una aplicación en funcionamiento en servidores accesibles.
  • Escalado vertical = más recursos al mismo servidor.
  • Escalado horizontal = más servidores.
  • IaaS = gestionas SO y aplicaciones.
  • PaaS = subes tu aplicación.
  • SaaS = solo usas el software.
  • DNS = nombre → IP.
  • LDAP = usuarios, permisos y directorios.
  • DIT = árbol de LDAP.
Errores típicos:
  • Confundir despliegue con programación.
  • Confundir DNS con LDAP.
  • Confundir FTP con SSH.
  • Separar mal los subdominios en ejercicios DNS.
  • Tomar verbos como entidades en los ejercicios de dominio.
