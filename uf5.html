UF5 - Documentación y Control de Versiones

📌 RESUMEN CLAVE

  • Ciclo de vida: requisitos → especificación → desarrollo → pruebas → despliegue → mantenimiento
  • Documentación: clara, estructurada, útil, actualizada
  • Git: sistema distribuido
  • Áreas: working directory → staging → repositorio
  • CI: pruebas automáticas | CD: despliegue

💻 COMANDOS GIT IMPORTANTES

Comando Función
git init Crea un repositorio nuevo
git clone Copia un repositorio remoto
git status Muestra el estado del repositorio
git add Prepara archivos para commit
git commit Guarda cambios (nueva versión)
git log Muestra historial
git diff Muestra cambios
git restore Deshace cambios
git reset Vuelve a versión anterior
git rm Elimina archivos
git mv Mueve o renombra archivos
git branch Gestiona ramas
git switch Cambia de rama
git checkout Cambia de rama o restaura
git merge Une ramas
git stash Guarda cambios temporalmente
git tag Marca versiones
git fetch Descarga cambios sin aplicar
git pull Descarga y aplica cambios
git push Sube cambios al remoto
