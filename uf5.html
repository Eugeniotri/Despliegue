Documentación Git Ramas Repositorio CI/CD

1. Ideas clave de la unidad

  • La documentación debe ser clara, ordenada, útil y actualizada.
  • Un proyecto de software suele pasar por: requisitos → especificación → desarrollo → pruebas → despliegue → mantenimiento.
  • La documentación reduce errores, facilita el mantenimiento y ayuda al trabajo en equipo.
  • Git es un sistema de control de versiones distribuido.
  • Permite guardar cambios, recuperar versiones, trabajar con ramas y colaborar con otras personas.

2. Lo que más cae de Git

  • Working directory: donde trabajas con los archivos.
  • Staging area: zona intermedia donde preparas los cambios.
  • Repositorio: donde quedan guardadas las versiones confirmadas.
  • commit: crea una nueva versión del proyecto.
  • branch: permite trabajar en paralelo sin tocar la rama principal.

3. CI y CD

  • CI (Integración continua): automatiza compilación y pruebas cada vez que hay cambios.
  • CD: puede significar entrega continua o despliegue continuo, según el contexto.
  • La idea general es automatizar tareas repetitivas para reducir errores y acelerar el desarrollo.

4. Comandos Git que debes memorizar

ComandoPara qué sirve
git initCrea un repositorio nuevo.
git cloneCopia un repositorio remoto en tu equipo.
git statusMuestra el estado actual del repositorio.
git addPasa cambios al staging area.
git commitGuarda una nueva versión de los cambios preparados.
git logMuestra el historial de commits.
git diffMuestra diferencias entre versiones o cambios.
git restoreDeshace cambios en archivos.
git resetRetrocede a estados anteriores o quita cambios del staging.
git rmElimina archivos del control de versiones.
git mvMueve o renombra archivos.
git branchCrea, lista o gestiona ramas.
git switchCambia de rama.
git checkoutCambia de rama o restaura contenido, según uso.
git mergeUne cambios de una rama con otra.
git stashGuarda cambios temporalmente sin hacer commit.
git tagMarca versiones importantes.
git fetchDescarga cambios remotos sin integrarlos.
git pullDescarga e integra cambios remotos.
git pushSube commits al repositorio remoto.

5. Trampas típicas de examen

  • git add no guarda definitivamente: solo prepara.
  • git commit sí guarda una nueva versión.
  • git fetch descarga, pero no integra.
  • git pull descarga e integra.
  • HEAD suele referirse al último commit de la rama actual.
  • Las ramas sirven para trabajar en paralelo.
  • MediaWiki es un sistema tipo wiki, no una herramienta de manuales man ni una etiqueta de Javadoc.
  • En páginas man, la sección 1 es la de comandos de usuario.
