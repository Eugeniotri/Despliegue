Resumen express

1. Idea general de la unidad

  • La transferencia de archivos en red normalmente sigue el modelo cliente-servidor.
  • Elementos básicos: cliente, servidor, protocolo de transferencia y sistema de archivos.
  • Protocolos importantes: FTP, FTPS, SFTP, TFTP, NFS, WebDAV, SMB/CIFS, Samba, rsync.
  • Se pueden montar directorios remotos como si fueran locales; esto hace la transferencia más transparente para el usuario.
Memoriza: cliente-servidor + nombre de los protocolos más importantes + que Samba es una implementación libre de SMB/CIFS.
FTP

2. Lo más importante de FTP

  • FTP no es seguro: credenciales y archivos viajan sin cifrar.
  • FTP usa 2 conexiones:
    • Control → puerto 21
    • Datos → puerto 20
  • El cliente solicita la conexión y el servidor responde y gestiona archivos.
  • Tipos de transferencia:
    • ASCII → para texto, con conversión de formato.
    • Binario → para todo lo que no sea texto (imágenes, pdf, zip...).
Trampa clásica: FTP no usa un solo canal, usa dos. Y SFTP no es lo mismo que FTPS.
Activo / Pasivo

3. Modos de conexión FTP

  • Modo activo: el cliente inicia la conexión de control, pero el servidor inicia la conexión de datos hacia el cliente.
  • Modo pasivo: el cliente inicia tanto la conexión de control como la de datos.
  • El modo pasivo evita problemas con cortafuegos del cliente, por eso es el más usado.
Memoriza: activo = servidor inicia datos. Pasivo = cliente inicia todo.
Códigos FTP

4. Códigos de respuesta FTP

  • 1xx → respuesta preliminar positiva
  • 2xx → finalización positiva
  • 3xx → hace falta más información
  • 4xx → error temporal
  • 5xx → error permanente

Ejemplos típicos: 220 servidor preparado, 230 login correcto, 331 pide contraseña, 530 autenticación incorrecta, 550 archivo inaccesible.

Accesos y clientes

5. Acceso al servidor FTP

  • Anónimo: sin usuario ni contraseña. Normalmente solo lectura.
  • Autorizado: requiere usuario y contraseña. Permisos por usuario.
  • Clientes FTP:
    • Modo comando → usa comandos como get, put, cd, lcd.
    • Modo gráfico → más cómodo, por ejemplo FileZilla.
    • Explorador de archivos → solución intermedia.
  • Los navegadores actuales ya no suelen permitir FTP.
FTPS y SFTP

6. Diferencias que más preguntan

  • FTPS = FTP sobre TLS/SSL.
  • SFTP = protocolo independiente que normalmente funciona sobre SSH.
  • SFTP usa 1 solo canal y normalmente trabaja por el puerto 22/TCP.
  • En SFTP la autenticación y el cifrado los aporta el canal seguro (normalmente SSH).
No confundas: FTPS ≠ SFTP. FTP seguro con TLS es FTPS. Protocolo distinto sobre SSH es SFTP.
OpenSSH, SCP, SSHFS

7. Parte práctica a recordar

  • OpenSSH es una implementación muy usada de SSH.
  • Incluye servidor y cliente SSH, además de SFTP y SCP.
  • SCP permite copiar archivos entre cliente-servidor e incluso entre dos servidores remotos.
  • SSHFS permite montar un sistema de archivos remoto como si fuera local.
  • SSHFS se basa en SFTP + FUSE.
  • ChrootDirectory en OpenSSH se usa para “enjaular” al usuario.
Muy importante: SSHFS = remoto como local. SCP = copiar con SSH. SFTP = gestionar archivos remotos de forma segura.
Lo mínimo para aprobar

8. Repaso final de memoria

  • FTP → no seguro, 2 conexiones, puertos 20/21.
  • FTPS → FTP + TLS/SSL.
  • SFTP → seguro, 1 canal, puerto 22, sobre SSH.
  • Activo → servidor inicia datos.
  • Pasivo → cliente inicia todo.
  • ASCII → texto. Binario → resto de archivos.
  • 4xx error temporal; 5xx error permanente.
  • Acceso anónimo suele ser solo lectura.
  • OpenSSH incluye SSH, SCP y SFTP.
  • SSHFS monta remoto como local.
