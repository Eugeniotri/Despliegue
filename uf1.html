Tema 1

Despliegue de aplicaciones web

El despliegue es el proceso de instalar, configurar y poner en funcionamiento una aplicación web en un servidor para que los usuarios puedan usarla.

Ciclo de vida del software

  • Análisis: se estudian las necesidades.
  • Diseño: se planifica la estructura de la aplicación.
  • Implementación: se programa la aplicación.
  • Despliegue: se instala y configura en servidores.
  • Implantación: pasa a producción.
  • Mantenimiento: correcciones y mejoras.

Ideas clave

  • El despliegue debe automatizarse para evitar errores humanos.
  • Las aplicaciones web se ejecutan en servidores, máquinas virtuales o contenedores.
  • Deben atender a muchos usuarios al mismo tiempo.

Escalabilidad

  • Escalado vertical: aumentar CPU, RAM o disco en un solo servidor.
  • Escalado horizontal: añadir más servidores para repartir la carga.
Tema 2

Alojamiento interno y computación en la nube

Alojamiento interno (in-house)

La empresa mantiene sus servidores dentro de sus propias instalaciones y se encarga de todo.

  • Hardware
  • Software
  • Red
  • Seguridad
  • Mantenimiento

Servicios típicos en in-house

  • DNS: traduce nombres a IP.
  • LDAP: centraliza usuarios y permisos.
  • Servicios web: Apache, Nginx, Tomcat, WordPress, Moodle.
  • FTP: transferencia de archivos.
  • Correo: SMTP, POP3, IMAP.
  • Conexión remota: SSH, VNC, RDP.

Problemas del modelo in-house

  • Coste alto.
  • Mantenimiento complejo.
  • Necesidad de personal especializado.
  • Riesgos por cortes eléctricos o fallos de red.

Computación en la nube

Permite usar servidores y servicios a través de Internet sin tenerlos físicamente en la empresa.

  • IaaS: infraestructura como servicio. Ejemplo: AWS EC2.
  • PaaS: plataforma como servicio. Ejemplo: Heroku.
  • SaaS: software como servicio. Ejemplo: Gmail, Google Docs.

DevOps

DevOps une desarrollo y operaciones para trabajar juntos durante todo el ciclo de vida del software, usando CI/CD.

Tema 3

Sistema de nombres de dominio (DNS)

El DNS traduce nombres de dominio en direcciones IP.

Conceptos básicos

  • Ejemplo: www.google.com → IP.
  • Está organizado como un árbol invertido.
  • El nombre completo se llama FQDN.

Niveles del DNS

  • Raíz: .
  • TLD: .com, .org, .es
  • Segundo nivel: google.com, gva.es
  • Subdominios: www.google.com, mail.google.com

Tipos de dominio

  • ccTLD: de país (.es, .fr)
  • gTLD: genéricos (.com, .org, .net)
  • lTLD: lingüísticos (.cat, .eus, .gal)

Resoluciones

  • Directa: nombre → IP
  • Inversa: IP → nombre

Zonas y servidores DNS

  • Zona directa y zona inversa.
  • Servidor primario: se edita la zona.
  • Servidor secundario: copia la zona.
  • Caché: guarda respuestas temporalmente.
  • Reenviador: envía consultas a otro DNS.

Registros DNS importantes

  • SOA: inicio de autoridad.
  • NS: servidor de nombres.
  • A: nombre a IPv4.
  • AAAA: nombre a IPv6.
  • PTR: resolución inversa.
  • MX: correo.
  • CNAME: alias.
  • TXT: texto.
  • SRV: ubicación de servicios.
Tema 4

LDAP y OpenLDAP

LDAP es un protocolo que permite centralizar la autenticación y autorización de usuarios en la red.

Qué gestiona

  • Usuarios
  • Grupos
  • Permisos
  • Servidores
  • Impresoras
  • Recursos de red

Funcionamiento

  • Modelo cliente-servidor.
  • Una aplicación consulta el servidor LDAP para validar usuario y contraseña.
  • Puede haber servidor maestro y esclavo.

Árbol DIT

La información se organiza en un árbol llamado DIT (Directory Information Tree).

  • dc: domain component
  • ou: unidad organizativa
  • uid: identificador de usuario
  • cn: nombre común

ObjectClass

  • Estructural: crea entradas reales.
  • Auxiliar: añade atributos.
  • Abstracta: clase base, como top.

Esquemas y LDIF

  • Los schemas definen clases y atributos.
  • LDIF es el formato de texto para importar y exportar datos.

OpenLDAP

OpenLDAP es la implementación libre más utilizada de LDAP en sistemas GNU/Linux.

Tema 5

Despliegue con contenedores

Los contenedores permiten aislar aplicaciones y dependencias para que funcionen igual en desarrollo, pruebas y producción.

Ventajas

  • Son ligeros.
  • Arrancan rápido.
  • Consumen menos recursos.
  • Facilitan el despliegue.
  • Mejoran la consistencia entre entornos.

Base técnica

  • cgroups: limitan recursos.
  • namespaces: aíslan procesos y recursos.
  • systemd: gestiona servicios y procesos.

LXC

LXC es una tecnología de contenedores en Linux y forma parte de la evolución de Docker.

Docker

  • Docker CE: versión comunitaria.
  • Docker EE: versión empresarial.

Elementos de Docker

  • Imagen: plantilla de solo lectura.
  • Contenedor: instancia en ejecución de una imagen.
  • Dockerfile: archivo para construir imágenes.
  • Registry: almacén de imágenes, como Docker Hub.
  • Docker CLI: cliente de comandos.
  • Docker Host: servidor donde se ejecutan contenedores.

Orquestación

Cuando hay varios contenedores, se usan orquestadores para coordinarlos.

  • Docker Compose: usa docker-compose.yml.
  • El archivo está en formato YAML.

Mini resumen final ultra rápido

  • Despliegue: poner una aplicación en producción.
  • In-house: infraestructura propia.
  • Cloud: infraestructura en la nube.
  • DNS: traduce dominios en IP.
  • LDAP: centraliza usuarios y autenticación.
  • Contenedores: aíslan aplicaciones y dependencias.
  • Docker: herramienta principal para contenedores.
  • Docker Compose: gestiona varios contenedores.
