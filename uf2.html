Resumen de la UF2

Objetivo: repasar lo esencial de servidores web, HTTP, HTTPS, TLS, hosts virtuales, hosting y dominio.

1. Servidores web y arquitectura

  • Internet funciona sobre TCP/IP.
  • La World Wide Web usa HTTP para transferir información.
  • En la arquitectura cliente-servidor, el cliente pide recursos y el servidor los busca y los sirve.
  • El servidor suele estar siempre encendido y en escucha.
  • En una arquitectura de tres niveles intervienen: cliente, servidor de aplicaciones y base de datos.
  • Ejemplo típico de tres niveles: WordPress o cualquier CMS dinámico.
  • Un proxy actúa como intermediario y puede cachear, filtrar, autenticar y balancear carga.
Memoriza: cliente = pide, servidor = responde, tres niveles = cliente + servidor de aplicaciones + base de datos.

2. LAMP

  • LAMP = Linux, Apache, MySQL, PHP.
  • Es un entorno clásico para alojar webs dinámicas.

3. HTTP

  • HTTP es un protocolo de capa de aplicación.
  • Normalmente usa TCP y escucha en el puerto 80.
  • Es un protocolo sin estado (stateless).
  • Para mantener información entre peticiones se usan técnicas como cookies.
  • En HTTP/1.1 aparecen las conexiones persistentes: varias comunicaciones HTTP sobre una sola conexión TCP.
  • En HTTP/2 hay mejoras como binario, compresión de cabeceras, multiplexación y server push.
Trampa típica: HTTP no es con estado. Si una pregunta dice que HTTP recuerda al usuario por sí mismo, es falsa.

4. URL, URI y recursos

  • URI = identificador de recurso.
  • URL = localiza un recurso mediante su dirección.
  • Estructura básica: esquema://host:puerto/ruta?parámetros.
  • El esquema suele ser http o https.

5. MIME, cabeceras y cuerpo

  • Los tipos MIME indican el tipo de contenido: tipo/subtipo.
  • Ejemplos importantes: text/html, application/json, image/png, application/pdf.
  • Un mensaje HTTP tiene cabecera y, opcionalmente, cuerpo.
  • La cabecera contiene metadatos; el cuerpo contiene el recurso o los datos enviados.
  • Content-Type indica el tipo MIME del cuerpo.
Memoriza: MIME describe el contenido, no la extensión del archivo. El navegador interpreta el cuerpo según Content-Type.

6. Métodos HTTP

  • GET: obtener recurso.
  • HEAD: solo cabecera, sin cuerpo.
  • POST: enviar datos para procesarlos.
  • PUT: crear o actualizar recurso.
  • DELETE: borrar recurso.
  • PATCH: modificar parcialmente.

7. Códigos de estado HTTP

  • 1xx: informativos.
  • 2xx: satisfactorios. Ejemplo: 200 OK.
  • 3xx: redirecciones. Ejemplo: 301 Moved Permanently.
  • 4xx: error del cliente. Ejemplos: 400, 403, 404.
  • 5xx: error del servidor. Ejemplos: 500, 503, 504.

8. HTTPS y TLS

  • HTTPS no es un protocolo totalmente nuevo: es HTTP sobre TLS.
  • Antes se usaba SSL; hoy se usa TLS.
  • HTTPS aporta autenticación, confidencialidad e integridad.
  • El puerto por defecto de HTTPS es el 443.
  • TLS 1.3 se publicó en 2018 en la RFC 8446.

9. Cifrado y PKI

  • Cifrado simétrico: misma clave para cifrar y descifrar. Es rápido.
  • Cifrado asimétrico: clave pública y clave privada. Es más seguro pero más lento.
  • TLS usa cifrado híbrido: asimétrico para el arranque y simétrico para la sesión.
  • La PKI permite confiar en las claves públicas.
  • Un certificado digital contiene la clave pública del propietario firmada por una autoridad certificadora.
  • La clave privada nunca se comparte.
Trampa típica: el certificado no contiene la clave privada. Si una opción dice eso, es incorrecta.

10. TLS: Handshake y registro

  • El protocolo Handshake negocia claves, parámetros y autenticación.
  • El protocolo de registro aplica esos parámetros para proteger la comunicación.

11. Hosts virtuales

  • Los hosts virtuales permiten alojar varios sitios web en un mismo servidor.
  • Tipos: basados en IP, basados en nombres y basados en puertos.
  • El más común es el basado en nombres: varios dominios en una sola IP.
  • El DNS resuelve dominio → IP.
  • El host virtual del servidor resuelve dominio → carpeta web.
  • En Apache, los hosts virtuales suelen configurarse en /etc/apache2/sites-available.
  • Para activar un sitio se usa a2ensite; para desactivarlo, a2dissite.
Frase clave: el DNS lleva al servidor y el host virtual decide qué carpeta servir.

12. Hosting y dominio

  • Dominio = nombre de la web.
  • Hosting = servidor y espacio donde están los archivos.
  • Normalmente se pagan por separado, aunque una misma empresa puede ofrecer ambos servicios.
  • Las empresas de hosting suelen ofrecer varios planes y un panel de control.
Muy importante para examen: dominio y hosting no son lo mismo.
